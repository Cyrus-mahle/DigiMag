Object Oriented Programming

Object Oriented Programming (OOP) is Programming methodology that is based on real world approach, This approach allows us to view problems we want to solve in object perspective, Object is an instance of class, it has properties or characteristics and functionalies or behaviors that is why it is based on world entity, While class is a user-defined data type which is a collection of objects.

OOP is a technique for writing, clean, organized and mantainable programs using the 4 features of OOP.
4 features of OOP are as follows:

  • Encapsulation Refers to the binding of properties(variables) and functionalies(functions or methods) into a single unit this also allows data hiding by declaring variables private and functions public.
  • Abstraction Refers to process of hiding how things works showing how to use them, providing a clear separation between the external usage of an object and its internal implementation.
  • Polymorphism: Polymorphism is a feature whereby an object has many forms, A real-life example of polymorphism, a lecturer at the same time is a mother, a wife, a woman. Poly means many while morphism means form
  • Inheritance is a mechanism in object-oriented programming where a new class (child or subclass or derived class) is created based on an existing class (parent or superclass or base class), inheriting its attributes and behaviors. This models an "is-a" relationship, where the subclass is a specialized version of the superclass.e.g I have base class Person and derived class Doctor, doctor is still a Person
    Composition: One or more members of a class are objects of another class for example every car has engine regardless of its type of engine and every vehicle has wheels regardless of number of wheels, Has-a-relationship


Its not necessary to use OOP all the time when you solve problems, some problems may be solved perfectly with the structured design approach, for example if the problem requires manipulating of sets of numbers with mathematical equations and functions, this can be solved using structured design.
While problems such as billing system for a hospital may require OOP.
